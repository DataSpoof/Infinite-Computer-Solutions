Pep 8 coding standard

prompt(this is python code write using solid and pep-8 coding standard)

* Use 4 spaces for indentation.
* Never mix tabs and spaces.
* Keep line length within 79 characters.
* Leave 2 blank lines between top-level functions and classes.
* Leave 1 blank line between class methods.
* Place imports at the top of the file.
* Group imports as: standard library, third-party, local modules.
* Use one import per line.
* Use `snake_case` for variables, functions, and methods.
* Use `PascalCase` for class names.
* Use `UPPER_CASE` for constants.
* Prefix non-public members with a single underscore (`_`).
* Use meaningful and descriptive names.
* Write one statement per line.
* Add spaces around operators.
* Add a space after commas.
* Do not add spaces inside brackets or parentheses unnecessarily.
* Use `is` / `is not` when comparing with `None`.
* Use `==` for value comparisons.
* Avoid `== True` and `== False`.
* Write docstrings for modules, classes, and functions.
* Keep comments clear and relevant.
* Avoid unnecessary comments.
* Keep functions short and focused on a single responsibility.
* Avoid deep nesting.
* Prefer list comprehensions when they improve readability.
* Use f-strings for string formatting.
* Use context managers (`with`) for file handling.
* Avoid wildcard imports (`from module import *`).
* Remove trailing whitespace.
* End every file with a single newline.
* Follow consistent formatting throughout the project.
